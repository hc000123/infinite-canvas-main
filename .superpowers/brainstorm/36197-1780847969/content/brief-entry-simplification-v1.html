入口收口规则

保留一个主入口 单集页只保留“生图需求 / Brief 审核台”，承接所有 Agent 产出的 Brief 草案。
其他地方不再新增 Brief 资产提取、设定库、分镜包只显示“已有 Brief / 打开 Brief / 创建需求”，不展示完整 Brief 表单。
人工不是主路径 手动新增 Brief 放到更多菜单，默认通过 Agent 草案进入审核。

调用关系

资产提取 写入生图需求，触发 Brief Agent 生成草案。
设定库 作为上下文和回流对象，不再作为单独 Brief 工作台入口。
分镜生产包 发现缺参考图时跳转到对应生图需求。
画布 只引用已生成主参考图，不再承担 Brief 编写。
建议：把“生图 Brief 工作台”改名为“生图需求审核台”或“美术 Brief 审核台”。主路径是 Agent 生成草案 -> 用户审核 -> 生成参考图 -> 回流项目资产库。项目页、资产提取、设定库、分镜和画布不再各自放完整 Brief 入口，只保留状态和跳转。
