# 迭代计划

本文档记录后续功能的整体规划、执行顺序和验收边界。当前工具已经从“画布生成工具”进入“AI 短剧创作工作台”阶段：项目是总入口，画布负责自由编排，剧本和分镜负责结构化生产，素材库负责资产沉淀，提示词仓库负责复用方法，已有 Agent 负责本地建议和动作预览。

已完成但仍需要真实页面确认的事项继续记录在 `docs/pending-test.md`。本文档只记录接下来要做什么、为什么做、先后顺序和不做什么。

## 当前基线

- 当前版本：`v0.2.70`。
- 已具备项目工作台、画布项目预设、剧本管理、分镜管理、设定库、提示词仓库、素材自动归档、素材去重、素材版本历史、素材版本引用锁定、生成队列、剪辑包导出、本地 Agent 工作台和 Agent Skill registry。
- 火山方舟 / Seedance 的本地直连边界已收口：本地直连只保留 OpenAI 兼容能力，火山方舟 / Seedance 企业能力走云端渠道；火山素材加白仍由后台系统设置维护。
- 当前核心短板不再是“缺入口”，而是生产链路的 Agent 化收口：本集工作台已经把剧本、分镜头表、生成镜头组、资产绑定、生成状态和画布节点收进同一条主路径；下一阶段要先建立统一 Agent 设置中心，再把剧本拆解、资产提取、生图需求、视频提示词和镜头组视频节点接入同一套可配置、可确认的 Agent 能力。

## 规划原则

- 项目优先：新能力优先挂在项目上下文下，避免继续形成分散工作台。
- 结果资产化：图片、视频、音频和后处理结果都要进入素材体系，并记录来源、提示词、模型、参数、任务和引用关系。
- 引用可控：画布、分镜、设定库默认锁定素材版本，不因素材更新自动改变旧项目结果。
- Agent 可确认：Agent 可以分析、建议、预览和生成动作草案，但不能绕过用户确认修改素材、画布、分镜、主版本或触发扣费生成。
- 云端后置但边界明确：本地能力先稳定数据模型；需要密钥、扣费、任务日志和多人共享的能力走后端云端渠道。
- 后处理不覆盖原视频：超分、字幕抹除、补帧、增强等结果都作为新素材版本或派生素材保存。
- 文档分层：路线图写在本文档；已实现待测写在 `docs/pending-test.md`；用户确认稳定后再沉淀到 `docs/features.md`。
- 后续执行指令必须标注模型推理程度：`中`、`高`、`超高`，用于控制额度消耗。

### M6.6.3：分镜头表与生成镜头组

目标：把本集剧本拆成可编辑的分镜头表，再由用户把连续分镜头组合成可生成的视频单元。生成镜头组是视频生成和生成表管理的基本单位，必须不超过 15 秒、同一个场次、不跨场景。

状态：已实现，待页面验收。当前分镜管理抽屉已新增“分镜头表 / 生成表”，可从已绑定本集剧本的画布生成分镜头草案、人工编辑 / 删除 / 排序、选择连续镜头组合生成镜头组、编辑视频提示词和绑定素材 / 设定；在具体画布中可确认“打组加入画布”，只创建提示词节点、参考素材节点和视频生成配置节点，不自动触发真实生成。

核心概念：

- 分镜头：剧本拆分后的单个镜头，包含画面描述、人物动作、对白、情绪、景别、时长、场次和场景。
- 生成镜头组：由一个或多个连续分镜头组合而成，用于生成一个视频段落。
- 生成表：按生成镜头组管理提示词、资产引用、音频引用、状态、任务和结果。

交付内容：

- 新增或增强分镜头表视图，支持按本集剧本生成分镜头草案。
- 受控 Agent 可把本集剧本拆成分镜头表草案；用户必须能逐条编辑、删除、排序和补充。
- 用户可选择多个连续分镜头组合成生成镜头组。
- 生成镜头组校验规则：总时长不超过 15 秒、同一个场次、不跨场景。
- 生成镜头组支持生成或编辑视频提示词。
- 生成镜头组支持绑定图片资产、音频资产、参考视频和设定库条目。
- 生成表展示每个生成镜头组的提示词、资产、状态、taskId、结果素材和主版本。
- 生成镜头组可打组加入画布，画布展示节点关系，生成表负责管理结构化状态。

流程关系：

```text
本集剧本
  -> 分镜头表草案
  -> 人工修分镜头
  -> 选择连续分镜头
  -> 生成镜头组
  -> 生成视频提示词
  -> 绑定资产图 / 音频 / 参考视频
  -> 加入画布生成
  -> 结果回流生成表 / 素材库
```

验收标准：

- 用户能从一集剧本生成分镜头表草案。
- 用户能独立编辑每个分镜头内容。
- 不能把跨场景或总时长超过 15 秒的分镜头组合成生成镜头组。
- 生成表能看到每个生成镜头组的提示词、资产、状态和结果。
- 画布仍展示生成节点和关系，但不承担完整表格管理职责。

不做事项：

- 不做无确认自动生成视频。
- 不把跨场景内容强行合并为一个生成镜头组。
- 不要求生成镜头组替代分镜头表。

### M6.7：生图 Brief 工作台

目标：把生图从“直接写提示词”升级为“先完成导演和美术思考，再生成图片”。Brief 工作台承接剧本资产拆解、设定库、分镜、参考图和提示词仓库，帮助用户在生成场景图、角色图、道具图、分镜氛围图前补齐关键信息、检查质量、拼装提示词，并把 Brief 快照和结果一起归档。

模型推理程度：

- 第一版制作使用 `高`：需要跨结构化 Brief 数据、store、表单、校验、提示词拼装、生成链路和素材 metadata 归档。
- 不使用 `超高` 做第一版：自动深度读剧本、自动拆场景、自动推荐电影 / 广告参考、自动生成完整美术方案先放到后续增强。

产品模式：

- 标准模式：场景、角色、道具生成前必须填完整核心项，否则不能生成。
- 提醒模式：系统提示缺了什么，但允许继续生成；默认使用该模式。
- 自由模式：屏蔽标准检查，只保留普通提示词生成和归档。

入口规划：

- 项目详情可进入“生图 Brief 工作台”。
- 画布图片生成配置节点可打开 Brief 抽屉。
- 本集资产拆解条目可直接创建角色图、场景图、道具图或氛围图 Brief。
- 设定库中的角色、场景、道具可从对应条目创建 Brief。
- 分镜条目可基于当前分镜创建氛围图或参考图 Brief。

第一版交付内容：

- 新增场景图、角色图、道具图三套结构化 Brief 表单。
- Brief 支持关联剧本片段、分镜条目、设定库条目和参考素材。
- 支持标准模式 / 提醒模式 / 自由模式。
- 新增 brief validator，按类型检查缺项、参考图用途和质量风险。
- 新增 prompt builder，把结构化内容拼成最终生图提示词。
- 最终提示词允许人工修改，并保存原始结构化 Brief 与最终提示词。
- 接入现有图片生成节点或图片生成入口。
- 生成成功后，图片自动入库，并在素材 metadata 中保存 Brief 快照、最终提示词、参考图和来源。

场景图核心字段：

- 剧本片段、本场情绪、场景拆分、地点、时间、天气氛围。
- 功能区域、道具陈设、人物与站位。
- 主光、辅光、逆光、光质、光比。
- 构图方式：纵深、三分法、对称、前景中景后景。
- 参考图用途：构图参考、灯光氛围、场景风格、禁用参考。
- 风格参考：电影感、写实、农村、科技、梦幻、黑暗、压抑等。

角色图核心字段：

- 剧本角色片段、角色身份、年龄、气质、当场情绪。
- 服装、发型、妆造、表情、姿态、视线方向。
- 拍摄景别：半身、全身、特写。
- 面部光质、背景或所在场景、角色一致性参考图、禁止跑偏项。

道具图核心字段：

- 道具名称、剧情作用、所属角色或场景。
- 年代、材质、尺寸、磨损程度。
- 摆放位置、是否需要被人物手持或使用。
- 光影质感、构图方式、细节参考图。

第二阶段方向：

- 模型推理程度升级为 `超高`，用于更复杂的剧本理解和美术指导判断。
- 粘贴剧本后自动分析情绪、人物、场景、道具。
- 自动建议是否拆成 2-3 个场景。
- 自动推荐需要哪些参考图类型。
- 自动生成初版 Brief，用户只需要修改和确认。

第三阶段方向：

- 每场戏形成场景资产包。
- 角色、道具、场景互相关联。
- 出图版本对比。
- 可导出为美术设定表、生图提示词表、分镜资产表。

验收标准：

- 用户能从场景 Brief 填写核心信息并生成最终提示词。
- 提醒模式下缺少关键字段会提示，但允许继续生成。
- 标准模式下缺少关键字段不能生成。
- 参考图必须能标注用途，并写入 Brief 快照。
- 生成成功图片能自动入库，素材详情能追溯 Brief、提示词和参考图。

不做事项：

- 第一版不接复杂 Agent 编排。
- 第一版不做全自动剧本理解。
- 第一版不做独立于项目体系之外的孤立工具。
- 第一版不替代现有设定库，只扩展设定库可承接的结构化 Brief 数据。

目标工作模式：

- 剧本驱动生产：新建画布时导入或绑定一集剧本；一集对应一个画布；本集工作台把剧本拆成可编辑的分镜头表；用户逐条修分镜头；再把同一场次、连续、总时长不超过 15 秒的分镜头组成生成镜头组；生成镜头组生成 / 编辑视频提示词，绑定图片、音频、参考视频和设定库资产；最后把生成镜头组导入画布执行视频生成，并把结果回流到生成表和素材库。

- 资产生产与复用：生图工作台和素材库负责制作、沉淀、筛选、版本化和复用角色图、场景图、道具图、氛围图等资产；资产既可以来自剧本拆解 Brief，也可以来自独立生图；进入分镜头组时必须能明确引用版本和来源。

- 分镜头表是否能作为主工作流入口，让用户看出下一步该修镜头、组镜头、补资产还是生成。

### M6.9：剧本到资产、镜头组和视频节点的 Agent 化工作台

目标：在 M6.8 本集工作台基础上，把剧本导入、资产提取、分镜草案、视频提示词和镜头组加入画布统一接入可配置 Agent。Agent 先通过“Agent 设置中心”维护提示词、变量、JSON 输出结构和写入策略；各工作台只负责调用、预览、确认写入，不再到处散落 Agent 提示词。

- Agent 设置必须独立成中心，不应散落在剧本、资产、生图、分镜各个工作台里。

目标流程：

```text
Agent 设置中心
  -> 配置资产提取 / 分镜导演 / 生图 Brief / 视频提示词 Agent
  -> 导入或绑定本集剧本
  -> 资产提取 Agent 输出资产 JSON 草案
  -> 用户确认写入本集生图需求
  -> 生图需求复用 Brief / 生图链路生成角色、场景、道具、氛围参考
  -> 分镜导演 Agent 输出分镜头表草案
  -> 用户编辑分镜头与生成镜头组
  -> 视频提示词 Agent 输出镜头组提示词草案
  -> 用户确认后加入画布为视频生成配置节点
  -> 绑定参考图 / 音频 / 视频并生成
  -> 结果回流本集工作台
```

- 如果是生成前标准化、Brief 表单、质量检查、提示词拼装，优先放入 M6.7 生图 Brief 工作台。
- 如果影响素材、版本、引用、导入导出、资产安全，优先放入 A 线或 C 线。
- 如果涉及云端文件、多人共享、权限、对象存储，优先放入 C 线。
- 如果是本机执行器、CLI、桌面端本地能力，暂不进入近期规划；先记录为远期想法，等主工作台生产闭环稳定后再评估。
- 如果是 Agent 分析、建议、质检、调度，需要区分两类：统一配置、草案预览、用户确认写入属于 M6.9 近期规划；自主调度、自动多步执行、自动扣费生成仍进入暂缓池。
- 剧本拆分镜头表第一版属于 M6.6.3 的受控草案生成能力；M6.9 起升级为可配置分镜导演 Agent 草案，但仍不纳入自主 Agent 编排。
- 所有会写入数据、扣费、生成视频、覆盖版本或删除内容的 Agent 能力，必须保留用户确认步骤。

## 指令推理程度规则

后续给开发指令时必须附带模型推理程度，帮助控制额度消耗：

- `中`：文档整理、简单 UI 调整、单文件工具函数、小范围筛选或展示逻辑。
- `高`：跨多个前端模块的数据结构、store、表单、校验、metadata 写入、生成链路接入。
- `超高`：后端表结构、云端对象存储、任务账本、权限、同步冲突、复杂跨端链路。

默认优先使用 `中`。只有涉及跨模块架构判断、数据一致性、云端安全或长链路生成时才提升到 `高` 或 `超高`。

## 暂缓池

- Agent 进阶编排：生成调度 Agent、剪辑顾问 Agent、合规 Agent、自主多步执行、自动扣费生成暂不进入近期规划；Agent 设置中心和可确认草案 Agent 已进入 M6.9。
- 即梦 CLI 本机版：本机执行器、CLI 队列、CLI 与主工作台桥接暂不进入近期规划。
- M7.5 视频后处理工具箱：火山视频超分、字幕抹除、补帧、增强等后处理能力暂不进入近期规划，避免过早牵出后端任务、扣费、上游接口和素材派生复杂度。
