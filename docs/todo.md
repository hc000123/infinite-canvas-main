# 迭代计划

本文档记录后续功能的整体规划、执行顺序和验收标准。近期主线是把当前工具从“画布生成工具”升级为“AI 短剧创作工作台”：画布负责自由编排，素材库负责沉淀结果，提示词仓库负责复用方法，分镜管理负责组织镜头，Agent 负责分析、建议、拆解、质检和整理。

## 规划原则

- 先打数据底座，再做复杂界面：素材去重、生成信息、项目预设、分镜引用关系要先稳定。
- 所有生成结果都要资产化：图片、视频、音频生成成功后应能进入“我的素材”，并记录来源、提示词、模型、参数和引用关系。
- Agent 入口集中，能力分开：用户看到统一的“短剧 Agent 工作台”，内部按资产、提示词、分镜、质检、调度等专职能力拆分。
- Agent 写入必须可确认：第一阶段只做建议、预览和用户确认后应用，不开放无确认自动生成、自动删除、自动覆盖主版本。
- 画布仍是自由创作空间：分镜、素材和 Agent 不能替代画布，只给画布提供结构化输入和结果归档。
- 云同步后置：本地素材、分镜和生成信息模型稳定后，再设计云端文件表、素材表和同步冲突策略。
- 共享资产库采用云端对象存储为主、本地缓存为辅：数据库只保存素材元数据、权限、版本和引用关系，图片/视频/音频原文件进入 S3 兼容对象存储或自建 MinIO。

## M5.0：数据底座

目标：先补齐后续所有功能都会依赖的基础数据结构，避免每个模块各自保存一套来源信息。

状态：已实现，待页面验收。当前已新增 `addAssetOnce`、素材去重指纹、metadata 合并逻辑，并改造现有加入素材入口；后续继续基于该能力接入生成结果自动归档。

### 交付内容

- 新增统一素材写入入口，例如 `addAssetOnce` 或 `upsertAsset`。
- 图片、视频、音频素材增加稳定的去重字段，例如 `fingerprint`。
- 图片和视频素材保存统一的 `generation` metadata。
- 画布节点、素材、分镜、生成任务之间建立可追溯引用。

### 建议数据结构

```ts
type AssetGenerationMetadata = {
  source:
    | "canvas"
    | "image-page"
    | "video-page"
    | "storyboard"
    | "agent"
    | "import";
  projectId?: string;
  projectTitle?: string;
  nodeId?: string;
  storyboardGroupId?: string;
  storyboardShotId?: string;
  prompt?: string;
  effectivePrompt?: string;
  model?: string;
  provider?: string;
  taskId?: string;
  actionType?: "generate" | "variant" | "edit" | "extend" | "continue" | "chat";
  references?: Array<{
    kind: "image" | "video" | "audio" | "text";
    label?: string;
    title?: string;
    assetId?: string;
    nodeId?: string;
    storageKey?: string;
    volcengineAssetId?: string;
  }>;
  config?: Record<string, unknown>;
  createdAt?: string;
};
```

```ts
type AssetFingerprint = {
  hash?: string;
  storageKey?: string;
  bytes?: number;
  mimeType?: string;
};
```

### 任务拆分

1. 为 `useAssetStore` 增加 `addAssetOnce/upsertAsset`。
2. 新增浏览器端 hash 计算工具，优先对 Blob 计算 SHA-256。
3. 扩展 `Asset.metadata`，统一保存 `generation`、`fingerprint`、`sourceRefs`。
4. 改造现有手动“加入我的素材”和导入逻辑，逐步改用去重入口。
5. 增加针对去重、metadata 合并、重复素材更新的单元测试。

### 验收标准

- 同一张生成图片重复保存时，“我的素材”只出现一条记录。
- 重复保存不会丢失新的来源节点、提示词和标签。
- 手动导入、画布保存、图片工作台、视频工作台都能继续保存素材。
- 删除素材仍能触发现有媒体清理逻辑，不误删仍被画布或分镜引用的文件。

### 不做事项

- 不做云端文件 hash。
- 不做跨设备去重。
- 不改现有本地素材存储方式。

## M5.1：生成素材自动归档

目标：画布生成成功的图片和视频自动进入“我的素材”，不再长期只堆在画布节点上。

状态：已实现，待页面验收。当前画布图片和视频生成成功后已静默调用 `addAssetOnce` 自动入库，并写入 `generation` metadata；失败路径不入库。

### 交付内容

- 画布图片生成成功后自动入库。
- 画布视频生成成功后自动入库。
- 视频素材保留生成提示词、实际提交提示词、模型、供应商、Endpoint、taskId、生成方式和来源节点。
- 自动入库使用 M5.0 的去重入口，不能直接无条件 `addAsset`。

### 任务拆分

1. 在图片生成成功路径中构造 `AssetGenerationMetadata`。
2. 在视频生成成功路径中构造 `AssetGenerationMetadata`。
3. 生成结果入库时写入 `fingerprint` 和 `generation`。
4. 如果素材已存在，只追加来源和更新时间，不新增重复卡片。
5. 自动入库成功后给用户轻提示，避免打断生成流程。

### 验收标准

- 画布生成单张图片后，“我的素材”出现对应图片。
- 画布批量生成多张图片后，每张成功结果都入库。
- 画布视频生成成功后，“我的素材”出现对应视频。
- 同一节点重试生成相同文件时不重复添加素材。
- 入库素材能插回画布继续使用。

### 验收标准

## M5.3：画布项目预设

目标：新建画布时设定项目级创作参数，后续画布节点、分镜、视频生成和素材归档默认沿用这些预设。

状态：已实现，待页面验收。当前 `CanvasProject` 已新增可选 `preset` 字段，新建画布可设置分辨率、画幅、帧率、默认模型和供应商，生成配置节点与视频生成默认值已能读取项目 preset。

### 交付内容

- 新建画布弹窗支持设置项目分辨率、画幅比例、帧率、默认时长、默认模型和默认供应商。
- 项目预设保存到 `CanvasProject`。
- 新建生成配置节点、视频节点、分镜组时默认读取当前画布预设。
- 生成素材的 `generation.config` 记录来源项目预设。

### 建议数据结构

```ts
type CanvasProjectPreset = {

  ratio?: string;
  fps?: string;
  defaultDuration?: string;
  defaultImageModel?: string;
  defaultVideoModel?: string;
  defaultTextModel?: string;
  defaultVideoProvider?: "openai" | "volcengine-ark";

```

### 任务拆分

1. 扩展 `CanvasProject` 类型，增加 `preset` 字段。
2. 新建画布弹窗增加预设表单。
3. 提供常用预设：竖屏短剧、横屏短片、方形社媒、高清横屏、高清竖屏。
4. 新建节点和分镜时读取项目预设作为默认值。
5. 增加项目设置入口，允许后续修改预设。

### 验收标准

- 新建“竖屏短剧”画布后，视频配置默认是 `9:16`。
- 新建分镜条目时继承画布的画幅、时长、模型。
- 修改单个节点参数不会反向覆盖项目预设。

## M5.4：角色场景设定库

目标：短剧最怕角色、场景、服装、道具在不同镜头里漂移。需要在素材库和分镜之前增加“设定库”，把角色卡、场景卡、道具卡和一致性参考固定下来，供提示词、分镜、Agent 和视频生成复用。

状态：已实现，待页面验收。当前已新增项目级设定库本地 store，并将入口迁移到“我的素材”的项目管理区，支持按画布项目打开角色 / 场景 / 道具设定库；设定支持 character / scene / prop 三类增删改、绑定“我的素材”和填写提示词片段；生成素材 metadata 已预留 `productionBibleRefs`。

### 交付内容

- 新增角色设定、场景设定、道具设定三类项目级资料。
- 每个设定可以绑定“我的素材”中的参考图片、视频或文本。
- 角色设定记录外貌、年龄、身份、服装、发型、表情范围、禁忌变化和常用参考图。
- 场景设定记录地点、时代、空间布局、光线、色调、可复用镜头描述。
- 道具设定记录名称、外观、用途、出现分镜和参考资产。
- 分镜和提示词仓库可以引用这些设定，避免每个分镜重复写人物与场景基础描述。

### 建议数据结构

```ts

  name: string;
  description: string;

    assetId: string;

  }>;
  promptSnippets?: {
    positive?: string;

    consistency?: string;
  };

  updatedAt: string;
```

### 任务拆分

1. 新增项目级设定库 store，支持角色、场景、道具。
2. 新增设定库页面或项目侧栏入口。
3. 支持从“我的素材”绑定参考图片和文本素材。
4. 分镜条目可以引用角色、场景、道具设定。
5. 提示词仓库变量支持引用设定库字段，例如 `{角色A}`、`{主场景}`。
6. Agent 可读取设定库，检查分镜和生成素材是否偏离角色/场景设定。

### 验收标准

- 用户能为一个短剧项目创建角色卡和场景卡。
- 分镜条目能引用角色卡和场景卡。
- 使用提示词模板时能把角色/场景设定填入最终提示词。
- 生成素材 metadata 能记录用到了哪些角色、场景或道具设定。

### 不做事项

- 第一版不做复杂人物关系图。
- 第一版不做自动训练角色模型。
- 第一版不强制所有分镜都必须绑定设定。

## M5.6：剧本与集数管理

目标：分镜管理之前需要有更上游的“故事结构”。短剧通常按集、场、情绪钩子推进，需要能保存故事大纲、分集剧情、场次、对白和关键情绪点，后续再由人工或 Agent 拆成分镜。

## M5.9：AI 短剧 Agent 工作台

目标：把 Agent 从单一聊天助手升级为多个专职 Agent 协同的短剧生产工作台。用户看到统一入口，内部能力按职责拆分。

状态：已实现第一版，待页面验收。当前已新增统一“短剧 Agent 工作台”入口、本地 `AgentTask` store，以及资产管理员、提示词工程、分镜导演三个本地规则 Agent；写入动作仍需用户在任务卡片中确认，不接入真实 LLM、不自动生成视频、不自动扣费。

### 产品形态

- 用户入口统一：短剧 Agent 工作台。
- 用户界面只暴露一个主 Agent，不把多个专职 Agent 做成一排复杂按钮。
- 主 Agent 负责理解目标、拆任务、选择 skill、汇总结果和生成动作预览。
- 内部能力分开：资产、提示词、分镜、生成调度、质检、合规等 Agent 独立实现。
- 专职 Agent 的能力优先沉淀为 skill，由主 Agent 调度执行。
- 第一版已落地 3 个 Agent：资产管理员 Agent、提示词工程 Agent、分镜导演 Agent。
- 后续再接入质检 Agent、生成调度 Agent、剪辑顾问 Agent 和合规 Agent。

### Agent 角色规划

- 制片 Agent：读取项目预设、画布、素材、分镜状态，规划当前最该推进的任务。
- 编剧 Agent：把故事梗概、小说片段拆成短剧大纲、场结构、情绪钩子和结尾悬念。
- 分镜导演 Agent：把剧本拆成分镜条目，补充景别、镜头运动、画面描述、时长和节奏。
- 角色一致性 Agent：维护角色卡、服装、发型、道具和场景设定，检查分镜是否偏离设定。
- 资产管理员 Agent：推荐可用素材、去重、补标签、关联分镜。
- 提示词工程 Agent：复用提示词仓库模板，生成分镜头提示词、正向词和负向词。
- 生成调度 Agent：把确认好的分镜转换为画布节点组，估算模型、时长、成本和依赖。
- 质检 Agent：检查生成视频的提示词匹配度、角色一致性、画面稳定性、镜头衔接和返修方向。
- 剪辑顾问 Agent：根据分镜顺序和主版本视频生成粗剪清单、字幕草稿、BGM/音效建议。
- 合规 Agent：提醒真人肖像、版权、敏感内容、平台限制和 AI 生成内容标识风险。

### Agent 工作模式

- 辅助模式：只给建议和诊断。
- 预览模式：生成分镜、素材匹配、提示词或画布节点变更预览，用户确认后应用。
- 半自动模式：用户选择目标后，Agent 连续执行多个确认步骤。
- 批处理模式：对多个分镜或素材批量生成标签、提示词、质检结论。

### Agent 任务中心

- 新增任务卡片，展示 Agent 建议、影响对象、风险和确认按钮。
- 任务状态：待确认、执行中、已应用、已取消、失败、需要人工处理。
- 任务卡片关联画布项目、分镜组、分镜条目、素材、提示词和生成任务。
- 所有写入动作需要有预览和操作记录。

3. 实现资产管理员 Agent：推荐素材、查重、补标签、关联分镜。（已实现第一版）

5. 实现分镜导演 Agent：从故事梗概或文本生成分镜条目，预览后写入分镜组。（已实现第一版）

### 验收标准

- Agent 生成的分镜先展示预览，用户确认后才写入。

## M5.10：Agent Skill 系统

目标：把 Agent 能力从“聊天提示词”收口为可版本化、可测试、可组合的 skill 工具箱。主 Agent 负责调度，skill 负责专业能力，动作预览负责落地前确认。

状态：已实现第一版，待页面验收。当前已新增本地 `AgentSkill` 协议和内置 skill registry，并把 M5.9 的资产、提示词、分镜核心能力拆成 4 个本地规则 Skill；后续再扩展 LLM Skill、用户自定义 Skill 和插件化能力。

### 核心原则

- Skill 是 Agent 的工具箱，不是另一个聊天机器人。
- 每个 skill 都要有清晰输入、清晰输出、版本号和启用状态。
- Skill 默认不直接写画布、素材、分镜或任务队列；需要写入时必须输出 action preview，由用户确认后应用。
- 本地规则 skill 优先，用于低成本、稳定、可测试的检查和整理。
- LLM skill 后置，用于提示词优化、分镜拆解、素材理解等更开放的任务。
- 用户自定义 skill 后置，不在第一版做插件市场。

### 架构分层

- 主 Agent：理解用户目标，读取画布、素材、分镜、任务状态，决定调用哪些 skill。
- Skill Registry：登记内置 skill 和用户启用状态，提供版本、分类、输入输出声明。
- Skill Runner：执行本地规则或 LLM skill，统一返回结构化结果。
- Action Preview：把 skill 产出的写入建议转成可预览、可确认、可撤销的动作。
- 业务落点：用户确认后再写入画布、素材、分镜、提示词仓库或生成队列。

### Skill 类型规划

- `asset.gap_check`：资产缺口检查，已实现第一版。
- `asset.reuse_duplicate_scan`：素材复用/重复检测，已实现第一版。
- `prompt.storyboard_completion`：分镜提示词补全，已实现第一版。
- `storyboard.scene_to_draft`：剧本场次转分镜草案，已实现第一版。
- `prompt.review.local`：本地提示词自审，后续 M6.1 接入。
- `prompt.optimize.seedance`：按 Seedance 规则优化提示词，输出修改建议或候选版本。
- `storyboard.generate`：把剧本或故事片段拆成分镜列表。
- `asset.tagging`：给素材补标签、用途、角色/场景关联。
- `shot.continuity`：检查上下镜头衔接、角色一致性和空间连续性。
- `video.qc`：对生成视频做可用性结论和返修建议。
- `generation.schedule`：检查分镜是否满足批量生成条件，输出队列预案和成本估算。

5. 有写入建议的 skill 复用现有动作预览和确认机制。（已实现第一版）

### 验收标准

- 后续新增 skill 时不需要改主 Agent 的核心对话流程，只需要注册和声明输入输出。

### 不做事项

- 第一版不允许用户上传可执行代码作为 skill。

## M6.1：生成前提示词自审

目标：在用户点击“生成视频”之前增加一个轻量质检层，先用本地规则检查提示词是否存在明显会导致 Seedance 结果跑偏的问题。第一版只提醒，不强制拦截，不调用大模型，不消耗额度。

状态：已实现第一版，待页面验收。当前已新增本地提示词自审纯函数、视频设置开关和画布视频生成前确认弹窗；用户仍可选择继续生成，自审不调用模型、不创建云端任务、不扣额度。

### 产品原则

- 用户可选择开启或关闭生成前自审。
- 默认是提醒机制：发现问题时展示自审结果，用户可以返回修改，也可以仍然生成。
- 第一版只做本地规则检查，不创建云端 `ai_tasks`，不扣云端额度。
- 自审配置不同于模型渠道配置，本地直连和云端渠道都可以使用同一套本地自审。
- 自审结果只辅助判断，不自动改写提示词，不替用户提交真实生成任务。

### 检查范围

- 基础完整性：主体、动作、场景、镜头、光线或风格是否缺失。
- 空间描述：左/右、前/后、远/近、室内/室外等关系是否含糊或互相矛盾。
- 否定表达：是否有过多“不要、不能、没有、避免、不要出现”等容易干扰模型的否定词。
- 正向表达：是否缺少明确的正向画面描述，导致模型只能从否定词反推。
- 风格词：写实、动漫、3D、纪实、电影感等风格是否混杂冲突。
- 动作时长：动作数量和复杂度是否明显不适合当前 5s/10s/15s 视频时长。
- 参考素材：有图片、视频、音频输入时，提示词是否说明参考素材用途。
- Seedance 模式：生成、编辑、延长、续写、首帧、首尾帧等模式是否和提示词表达匹配。

### 交付内容

- 新增本地提示词自审工具函数，输出 `pass / warning / risk` 等级、问题列表和修改建议。
- 视频生成设置中新增“生成前提示词自审”开关。
- 画布视频节点点击生成时，先执行本地自审。
- 自审弹窗展示总体结论、问题类型、问题说明和建议改法。
- 弹窗提供“返回修改”和“仍然生成”两个操作。

### 建议数据结构

```ts
type PromptReviewLevel = "pass" | "warning" | "risk";

type PromptReviewIssue = {
  type:
    | "space"
    | "negative"
    | "positive"
    | "style"
    | "motion"
    | "reference"
    | "seedance";
  level: PromptReviewLevel;
  title: string;
  description: string;
  suggestion?: string;
};

type PromptReviewResult = {
  level: PromptReviewLevel;
  summary: string;
  issues: PromptReviewIssue[];
};
```

### 任务拆分

1. 新增 `canvas-prompt-review` 纯函数和单元测试。（已实现）
2. 给本地配置增加 `videoPromptReviewEnabled` 开关，默认开启提醒。（已实现）
3. 在视频节点生成链路中接入自审，确保真正提交 Seedance 前触发。（已实现）
4. 新增自审结果弹窗，问题提示词先展示结果，不直接创建任务。（已实现）
5. 用户点击“仍然生成”后复用原视频生成流程，点击“返回修改”不创建任务。（已实现）
6. 后续 M6.3 再规划 AI 深度自审，可使用云端模型代理或本地直连模型，但不纳入第一版。

### 验收标准

- 自审关闭时，点击生成保持原流程。
- 自审开启时，明显有问题的提示词会先弹出自审结果。
- 用户点击“返回修改”不会创建视频任务。
- 用户点击“仍然生成”才继续创建视频任务。
- 本地规则自审不调用模型接口，不新增云端 `ai_tasks`。
- 单元测试覆盖通过提示词、否定词过多、缺少主体动作、参考素材用途缺失、编辑/延长模式不匹配。

### 不做事项

- 第一版不自动重写提示词。
- 第一版不调用大模型做深度审核。

## 独立版本线 C1：即梦 CLI 本机版

目标：把即梦 CLI 规划成相对独立的本机执行器版本，而不是主工作台视频设置里的一个普通选项。它先独立解决本机 CLI 配置、任务执行、结果收集、日志和批量能力，后续再通过桥接协议接入“眨眼之间工作台”的画布、素材和分镜。

### 产品定位

- 主工作台云端版：商业云控路径，后端保存渠道 Key，统一扣费、限流和记账。
- 主工作台个人本地版：用户自己填写 Base URL / Key，前端或本地后端直连模型服务。
- 即梦 CLI 本机版：独立本机执行器，优先服务已有本机登录态、批量生成、CLI 专属能力和离线整理流程。
- 后续桥接模式：主工作台通过受控本机服务调用 CLI 本机版，但不把它混进普通云端渠道配置。

### 核心原则

- 先做独立版本，不急着塞进主工作台生成设置。
- 独立版本需要自己的配置、任务日志、输出目录、错误展示和结果管理。
- 主工作台只通过桥接协议读取 CLI 状态和结果，不直接拼命令、不直接管理 CLI 登录态。
- CLI 配置和 Seedance 云端渠道配置分开，不把 CLI 登录态、Cookie、Token 或本机路径上传到云端。
- Web 云控商业版默认不允许浏览器直接调用本机 CLI。
- CLI 生成结果同样要转存到本地素材或云端素材，并写入来源 metadata。
- CLI 任务同样要进入任务可观测体系：创建、排队、执行、下载、完成、失败、取消。
- 不把 CLI 输出当作黑盒成功；必须解析退出码、stdout/stderr、结果文件和错误信息。

### 适用场景

- 用户已经在本机配置好即梦 CLI，希望直接用本机能力生成视频。
- 用户想先用独立本机工具完成批量生成，再把结果导入主工作台整理。
- 桌面版批量生成分镜，希望绕开浏览器长连接限制。
- CLI 提供某些 Web API 暂未接入的生成能力或参数。
- 本地调试 Seedance / 即梦任务参数，方便开发阶段对比 API 和 CLI 输出。

```ts
type JimengCliConfig = {

  workingDirectory?: string;
  outputDirectory?: string;
  defaultModel?: string;
  defaultArgs?: string[];
  timeoutSeconds?: number;
};
```

```ts
type JimengCliTaskMetadata = {
  provider: "jimeng-cli";
  actionType: "generate" | "variant" | "edit" | "extend";
  projectId?: string;
  batchId?: string;
  commandPreview?: string;
  workingDirectory?: string;
  outputFile?: string;
  exitCode?: number;
  stdoutSnippet?: string;
  stderrSnippet?: string;
};
```

### 任务拆分

1. 调研即梦 CLI 安装方式、鉴权方式、命令参数、输出目录和错误格式。
2. 独立版新增本地 CLI 配置入口：启用状态、CLI 路径、输出目录、超时时间和测试命令。
3. 独立版新增 CLI runner，负责执行命令、限制工作目录、处理超时和收集日志。
4. 独立版提供任务列表和批量任务入口，支持单条生成、批量生成、失败重试和结果定位。
5. 设计主工作台到 CLI 本机版的桥接协议：提交任务、查询状态、读取结果、取消任务。
6. 把画布视频生成参数转换成中立任务描述，再由 CLI 本机版转换成 CLI 参数。
7. CLI 执行完成后扫描结果文件，写入本地媒体存储或导出素材包，记录 metadata。
8. 后续主工作台接入桥接后，可把 CLI 结果创建为新视频节点或新素材。
9. 批量生成队列后续可选择 CLI executor，支持并发数、暂停、失败重试和输出文件清理。

### 安全边界

- 主工作台前端不允许把任意字符串拼成 shell 命令执行；参数必须结构化组装并交给本机 runner。
- CLI runner 不执行用户输入的自定义脚本，只调用配置好的可执行文件。
- 工作目录和输出目录必须限制在应用允许范围内。
- stdout/stderr 入库或展示前要脱敏，避免泄露本机路径、Token、Cookie 或账号信息。
- 云端模式不下发也不保存用户本机 CLI 凭据。

### 验收标准

- 独立版能检测本机 CLI 是否安装，并给出明确成功/失败提示。
- 配置 CLI 后，用户能在独立版里发起一次视频生成，并看到任务状态。
- CLI 生成完成后能定位结果文件，并保存任务 metadata。
- 独立版能导出结果素材包，供主工作台导入。
- 后续桥接接入后，主工作台能从 CLI 本机版读取结果并创建视频节点或素材。
- CLI 执行失败时显示可读错误，不误标记为生成成功。
- CLI 任务不会新增云端供应商 Key 暴露风险。

### 不做事项

- 第一版不把 CLI 能力开放给纯浏览器云端用户。
- 第一版不支持用户编写任意命令模板。
- 第一版不承诺 CLI 参数和火山 Ark API 完全一致。
- 第一版不把本机 CLI 登录态同步到云端。
- 第一版不要求主工作台视频设置直接出现“即梦 CLI”选项。

## M7：素材资产中心增强

目标：把“我的素材”从收藏列表升级为创作资产中心。

状态：M7.0 分镜筛选与排序、M7 批量整理/删除/导入校验/火山加白批处理、素材来源链路和项目共享库本地底座第一版已实现，待页面验收。当前“我的素材”已支持按分镜组筛选素材，能同时找到该分镜组直接引用的参考素材、分镜生成结果和 generation metadata 记录到同一 `storyboardGroupId` 的素材；列表支持默认、最近更新、最近生成、创建时间和标题排序。已选择素材后可批量移动到指定文件夹、批量追加标签和批量删除，已有标签会保留并去重。素材压缩包导入会校验 `assets.json`、包来源、版本、素材类型和文件清单，避免坏包半导入。图片/视频素材可批量提交火山加白或批量刷新火山加白状态，复用现有单素材加白接口和元数据结构。项目上下文下可把本地素材标记加入项目共享库，并在项目视图中筛选“仅项目库 / 未入项目库”。

### 交付内容

- 批量移动文件夹、批量打标签、批量删除和批量导出。
- 按提示词、模型、供应商、生成方式、素材来源、分镜组、最近生成时间筛选或排序。
- 素材压缩包导入导出校验。
- 火山加白素材批量刷新和失败后重新提交。
- 素材详情展示更完整的来源链路。
- 页面结构预留“本地素材 / 项目资产库 / 全部”视图，后续 M7.1 接入共享项目素材。

### 验收标准

- 用户能快速找到某个分镜下的所有生成视频。
- 用户能批量整理同一项目的素材。
- 导入素材包不会重复导入已有素材。
- 用户能明确区分当前素材是本地私有素材，还是后续项目共享资产。

## M7.1：项目共享资产库底座

目标：把一个项目内共用的图片、视频、音频资产从浏览器本地素材升级为可多人访问的项目级共享资产库。第一版先解决“同一个项目的人能看到同一套素材”，不直接做复杂实时协同。

状态：本地 metadata 底座已实现，待页面验收。当前只在本地素材 `metadata.projectLibraries` 中记录项目库 membership、角色、同步状态和远端 ID 预留；“我的素材”在选择项目上下文后支持批量加入/移出项目库、按项目库状态筛选，并在卡片和详情中展示项目库标记。云端项目资产表、成员权限校验、对象存储上传和签名 URL 仍按下方规划后续实现。

### 产品原则

- 项目资产库挂在 `Project` 下，不替代本地“我的素材”，而是作为独立来源并可和本地素材互相复制。
- 文件原件存云端对象存储，后端数据库只保存元数据、权限、版本指针和引用关系。
- 对象存储接口优先做 S3 兼容抽象，后续可接阿里 OSS、腾讯 COS、火山 TOS、Cloudflare R2、AWS S3 或自建 MinIO。
- 预览和下载必须经过后端权限校验，前端拿临时签名 URL，不长期暴露公开裸链接。
- 本地端保留缩略图和最近使用文件缓存，缓存失效时可重新从云端拉取。

### 建议数据结构

```ts
type ProjectMemberRole = "owner" | "admin" | "editor" | "viewer";

type ProjectAsset = {
  id: string;
  projectId: string;
  folderId?: string;
  title: string;
  kind: "image" | "video" | "audio";
  tags?: string[];
  currentVersionId: string;
  createdBy: string;
  createdAt: string;
  updatedAt: string;
};

type ProjectAssetVersion = {
  id: string;
  assetId: string;
  versionNumber: number;
  fileKey: string;
  thumbnailKey?: string;
  bytes: number;
  mimeType: string;
  checksum?: string;
  changeNote?: string;
  createdBy: string;
  createdAt: string;
};
```

### 权限模型

- owner：管理项目、成员、权限和所有素材。
- admin：管理素材、文件夹、版本和成员邀请。
- editor：上传素材、替换版本、编辑标题标签、移动文件夹。
- viewer：预览、下载、引用素材，不能修改共享库。

### 任务拆分

1. 后端新增项目成员、项目资产、项目资产版本、项目文件夹表结构。
2. 增加对象存储配置和上传服务，第一版支持一种 S3 兼容实现，保留 provider 字段。
3. 新增项目资产上传、列表、详情、删除、下载签名 URL 接口。
4. 前端“我的素材”增加来源切换：本地素材、项目资产库、全部。
5. 项目资产库支持上传图片、视频、音频，展示缩略图、媒体信息、标签和文件夹。
6. 画布当前项目生成成功时，可选择自动归档到项目资产库或本地素材。
7. 从项目资产库插入画布时，节点记录 `projectAssetId` 和 `versionId`。

### 验收标准

- 同一个项目的成员能看到同一套项目资产。
- editor 上传素材后，viewer 刷新页面可以预览和下载。
- 非项目成员访问素材接口会被拒绝。
- 图片、视频、音频原文件不写进数据库，数据库只保存对象存储 key 和元数据。
- 画布插入项目资产后，节点能记录资产来源并继续预览。

### 不做事项

- 第一版不做实时多人编辑。
- 第一版不做跨项目资产复用市场。
- 第一版不自动把所有本地素材迁移到云端，迁移入口单独确认。
- 第一版不在没有引用计数前自动物理删除对象存储文件。

## M7.2：素材版本与更新动态

目标：让项目共享资产库可以追踪“谁在什么时候更新了哪个素材”，并允许查看历史版本、恢复版本，以及控制画布节点是否跟随素材最新版。

状态：本地版本历史第一版已实现，待页面验收。当前编辑已有素材并替换文本、图片、视频或音频内容时，会在素材 `metadata.assetVersions` 中记录初始版本和新版本；只改标题、标签、备注、文件夹或项目库 membership 不会产生版本。素材详情可展示本地版本历史，并允许恢复旧版本；恢复会切换当前素材内容和 `currentAssetVersionId`。云端项目资产版本表、成员更新动态、旧版本下载、画布引用固定版本/跟随最新版仍按下方规划后续实现。

### 产品原则

- `assetId` 稳定不变，替换文件时新增 `AssetVersion`，不直接覆盖旧文件。
- 画布默认引用固定版本：记录 `assetId + versionId`，避免别人更新素材后旧画布突然变化。
- 用户可手动切换为“跟随最新版”，素材更新后画布节点提示可同步。
- 更新动态按项目聚合展示，让成员能快速查看最近新增、替换、恢复、移动和标签变化。

### 交付内容

- 素材详情新增“版本历史”区域，展示 v1 / v2 / v3、上传人、更新时间、文件信息和更新说明。
- 素材卡片和详情显示当前版本号。
- 支持“替换为新版本”和“恢复此版本”。
- 支持查看项目资产库更新动态。
- 画布节点展示项目资产引用状态：固定版本、可更新、已是最新版、来源版本已删除或不可访问。

### 建议数据结构

```ts
type ProjectAssetActivity = {
  id: string;
  projectId: string;
  assetId?: string;
  versionId?: string;
  action:
    | "asset_created"
    | "version_uploaded"
    | "version_restored"
    | "asset_moved"
    | "asset_tagged"
    | "asset_deleted";
  actorId: string;
  summary: string;
  createdAt: string;
};

type CanvasAssetReferenceMode = "fixed-version" | "follow-latest";
```

### 任务拆分

1. 项目资产上传接口支持 `replaceAssetId`，替换时写入新版本和更新动态。
2. 素材详情展示版本列表，支持预览旧版本和下载旧版本。
3. 增加恢复版本接口：把 `currentVersionId` 指回旧版本，同时写入恢复动态。
4. 画布节点保存 `assetReferenceMode`，默认 `fixed-version`。
5. 打开画布时检查项目资产引用，如果共享库已有新版本，给节点显示“有新版本”提示。
6. 增加项目资产库“更新动态”视图，按时间展示成员操作。

### 验收标准

- 替换素材后，旧版本仍可查看和下载。
- 恢复旧版本后，素材当前版本号和预览能回到旧文件。
- 画布固定版本节点不会因为素材库更新而自动变图或变视频。
- 画布跟随最新版节点能看到新版本提示，并由用户确认同步。
- 更新动态能看到上传、替换、恢复、移动、打标签等操作记录。

### 不做事项

- 第一版不做像 Git 一样的版本差异对比。
- 第一版不做多人同时编辑冲突合并。
- 第一版不自动批量升级画布里所有引用。

## M7.5：视频后处理工具箱

目标：把火山视频超分、字幕抹除等能力接入到已生成视频和视频素材上，作为“后处理 / 修复 / 增强”链路，而不是继续塞进 Seedance 生成设置。

### 产品原则

- 后处理入口放在视频节点工具栏和视频素材详情中，不放进视频生成参数面板。
- 后处理结果生成新视频节点或新视频素材，不覆盖原视频。
- 原视频和结果视频之间记录来源关系，建议关系类型为 `postprocess` 或 `enhancement`。
- 云端商业版走后端代理、额度扣费和 `ai_tasks` 账本；个人本地版后续可按渠道能力支持本地直连。
- 后处理能力和 Seedance 生成渠道分开配置，避免用户误以为超分/字幕抹除使用的是同一个生成 Endpoint。

### 能力范围

- 视频超分：提升视频分辨率和清晰度，用于成片前统一质量。
- 字幕抹除：去除画面中已有硬字幕、错误字幕或影响二次创作的文字区域。
- 视频补帧：在火山接口支持时提升帧率，让动作更顺滑。
- 视频降噪 / 锐化 / 色彩增强：作为后续可选增强项。
- 局部修复 / 去水印：仅在接口和合规边界明确后再开放。
- 封面帧 / 关键帧提取：从视频中提取封面或参考帧，和现有截帧能力协同。

### 画布关系

```text
原视频节点
  -> 视频超分
      -> 超分后视频节点
  -> 字幕抹除
      -> 无字幕视频节点
  -> 补帧/增强
      -> 增强后视频节点
```

### 建议数据结构

```ts
type VideoPostprocessAction =
  | "upscale"
  | "remove_subtitle"
  | "interpolate"
  | "denoise"
  | "sharpen"
  | "color_enhance";

type VideoPostprocessMetadata = {
  actionType: "postprocess";
  postprocessAction: VideoPostprocessAction;
  relationType: "postprocess" | "enhancement";
  sourceVideoNodeId?: string;
  sourceAssetId?: string;
  taskId?: string;
  model?: string;
  provider?: string;
  config?: Record<string, unknown>;
};
```

### 后端账本规划

- `ai_tasks.kind` 继续使用 `video`。
- `ai_tasks.actionType` 增加 `upscale`、`remove_subtitle`、`interpolate` 等后处理动作。
- 任务记录保存源视频 URL、源素材 ID、目标动作、参数、上游 taskId、结果文件和错误信息。
- 长任务状态沿用 M4 视频异步任务生命周期：排队、处理中、成功、失败、失败返还、人工刷新。
- 成功结果转存为云端素材或本地素材，并和原素材建立来源关系。

### 任务拆分

1. 调研并封装火山视频超分、字幕抹除接口参数和状态查询。
2. 后端新增视频后处理代理服务，接入扣费、失败返还、任务日志和脱敏。
3. 前端视频节点工具栏新增“视频增强”入口，第一版提供超分和字幕抹除。
4. 素材详情新增“后处理”操作，结果保存为新视频素材。
5. 画布中后处理结果创建新视频节点，并记录 `postprocess` 关系 metadata。
6. 后处理任务完成后自动转存结果，失败时展示可读错误并支持刷新状态。
7. 后续接入批量处理：对一组分镜主版本统一超分或统一字幕抹除。

### 验收标准

- 用户能从已完成视频节点发起“超分”，完成后得到新视频节点，原视频不被覆盖。
- 用户能从视频素材详情发起“字幕抹除”，完成后得到新素材，原素材仍保留。
- 后处理任务写入后台 `ai_tasks`，能看到动作类型、源视频、状态、扣费和错误。
- 后处理失败时可刷新状态，失败返还不能重复执行。
- 后处理结果可继续作为普通视频素材插回画布或参与后续生成。

### 不做事项

- 第一版不做复杂批量工作流。
- 第一版不把后处理参数混入 Seedance 生成设置。
- 第一版不覆盖原视频文件或原素材。
- 去水印 / 局部修复能力需要先确认接口能力和合规边界后再开放。

## M9：Agent 进阶编排

目标：在 M5.9 的基础上，让 Agent 逐步支持更长链路的半自动执行。

### 交付内容

- 支持“一键补齐本集分镜”的多步骤确认流程。

## M10：云同步预研

目标：在本地素材、分镜和生成信息模型稳定后，再设计云端同步能力。

### 预研内容

- 画布 JSON 与媒体文件分离存储。
- `storageKey` 和 `fileId` 双轨兼容结构。
- 后端文件表、素材表、画布表、分镜表和引用计数。
- 旧本地数据迁移策略。
- 导入导出兼容策略。
- 多设备同步冲突策略。

### 不做事项

- 云同步完成前，文档和 UI 不承诺已支持多设备同步。
- 不把大体积 base64 写入画布 JSON。
- 不在没有引用计数方案前自动删除云端文件。
