# 上线验收总报告

本文档用于给当前 AI 短剧制作工具做上线前决策。它不是功能介绍，而是把已经验证的证据、未验证的风险和放行条件集中到一个入口。

## 总结论

当前结论：内测候选有条件通过，正式上线不通过。

可以继续给制作团队做本机内测，重点验证“项目 -> 分集 -> 本集生产流程 -> 承接画布 -> 素材复用”的真实使用体验。不能直接面向普通用户发布，也不能承诺云端长期保存、真实模型生产链路、macOS 普通下载安装包和真实生图 / 视频生成已经完成验收。

| 合格线目标 | 当前状态 | 判定 |
| --- | --- | --- |

| 中途不丢数据 | 本地浏览器路径和 Docker 挂载持久化通过 | 本机 / Docker 条件通过 |

| 制作人员愿意替代零散文档和聊天窗口 | 工具链路已具备承接能力，但真实制作意愿必须等真实 API Key 和至少一集人工试用后判断 | 未达最终线 |

## 已验证证据

- 项目 / 分集 / 本集生产流程：真实浏览器完成创建项目、导入分集、进入本集生产流程、创建并进入承接画布。
- mock 单集链路：导演分析、资产提取、Seedance 分镜三阶段均进入 `approved`，设定库写入 2 项，分镜表写入 2 条，承接画布创建 2 个视频配置节点。
- 失败恢复：后端不可用时运行分析会进入异常状态，错误原因可见，剧本、项目、分集和失败状态不会因创建承接画布而被覆盖。
- 关键素材复用：`/assets` 新增素材、刷新持久化、加入项目库、本地素材插入画布、固定版本 metadata 写入、远程素材库文本素材插入画布均已验证。
- 画布基础路径：素材弹窗、配置包导入、多文件上传、文件拖入、文本节点鼠标拖动、新增文本 / 视频节点均已抽样验证。
- Docker 本地部署：生产镜像、`/api/health`、healthcheck、Go / Next 进程联动、`/app/data` 挂载持久化和 `/api/uploaded-assets/...` 单端口公开路径均已验证。
- 桌面包：`.app` 和去隔离 DMG 副本可启动主流程，桌面 Go 后端和 Next 服务可达，退出后无残留监听。

## 未通过或未完成项

### 真实 API Key 单集闭环

当前只有 mock 模型链路通过。还需要用有效真实 API Key 跑同一集，确认：

- 导演分析能稳定输出可用讲戏和结构化摘要。
- 资产提取能写入真实可用的角色、场景、道具或服化道设定。
- Seedance 分镜能产出可执行分镜表和视频配置节点。
- 模型超时、空返回、JSON 异常或质量门不通过后可以继续重试，不需要重建项目或分集。

### 真实云服务器部署

本地 Docker 运行态通过，但真实云服务器仍需复测：

- 域名和 HTTPS 可访问。
- `https://域名/api/health` 返回 `ok`。
- 容器挂载 `/app/data` 后，重启或重建不会丢 SQLite、公开素材和提示词缓存。
- 公开素材必须实测 `https://域名/api/uploaded-assets/...` 能被外部平台访问。
- 生产环境已设置非默认 `ADMIN_PASSWORD` 和非默认 `JWT_SECRET`。

### macOS 正式分发

当前 DMG 内容可运行，但普通下载 / 微信分发路径未通过：

- 带隔离属性的 app 会被 `spctl --assess` 判定为 `rejected`。
- 可能触发 App Translocation，导致桌面服务未进入健康态。
- 正式分发前需要 Developer ID 签名和 notarization，并重新验证首次打开体验。

### 真实生图和视频请求

当前没有执行真实扣费生图或视频生成。还需要验证：

- 生图请求能进入队列或返回真实结果。
- 视频请求能进入队列或返回真实任务状态。
- 生成结果能归档到素材，并保留来源、模型、参数、提示词和引用关系。
- 失败后能看到失败原因并重试。

## 放行建议

| 场景 | 建议 |
| --- | --- |
| 本机内测 | 可以放行 |
| 给制作人员小范围试用 | 可以放行，但要说明真实模型、云同步和正式安装包仍在验收 |
| 云服务器正式部署 | 暂不放行，等真实域名 / HTTPS / 持久化 / 公开素材路径通过 |
| macOS DMG 对外分发 | 暂不放行，等签名和 notarization 通过 |
| 对外宣传“完整 AI 短剧生产闭环” | 暂不放行，等真实 API Key 单集闭环和真实生图 / 视频通过 |

## 最短补验路径

1. 准备真实 API Key，按同一集剧本重跑导演分析、资产提取、Seedance 分镜。
2. 在云服务器用 Docker 部署，配置非默认密钥和 `/app/data` 挂载。
3. 上传公开素材，确认公网 `/api/uploaded-assets/...` 可访问。
4. 配置 macOS Developer ID 签名和 notarization，重新打 DMG 并用带隔离属性的下载路径验证。
5. 运行一次真实生图和一次真实视频任务，确认生成、失败、重试、归档链路。
