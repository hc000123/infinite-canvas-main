# 全产品 UI 改造方案

## 目标

将当前用户侧界面统一为一套偏专业创作软件的暗色设计语言，暂定名为 Studio Dark。画布、项目工作台、素材库、生图、视频、提示词库和项目详情页应共享同一套视觉规则，避免画布单独变好看、其他页面仍保留旧风格。

本轮只做用户侧主界面方案。管理后台保留为第二阶段，只同步基础 Ant Design token，不做沉浸式创作台视觉。

## 当前问题

- 页面风格割裂：项目工作台偏 `cyan/slate`，画布偏 `stone`，提示词中心偏浅色点阵，视频页又是另一套暗色。
- 工具层级混乱：画布底部工具栏放入创建、管理、外观、危险操作，节点悬浮工具条在图片/视频节点上过长。
- 颜色缺少统一语义：青色、蓝色、石色、茶色在不同页面承担相似含义。
- 全局样式已有 `studio-shell`、`studio-panel`、AntD theme 和 workspace shell，但没有形成统一设计系统。

## 设计方向

采用 D3 暗色 AI 创作台方向：

- 近黑蓝灰背景，减少纯黑和棕灰。
- 主色使用克制信号蓝，仅用于选中、生成、焦点和主操作。
- 成功、任务完成、可用状态使用柔绿色。
- 危险操作使用柔红，默认隐藏或弱化。
- 面板低对比、轻边框、少阴影。
- 圆角控制在 8-12px，节点和媒体预览可略大。
- 不使用霓虹、赛博、发光背景、大渐变和装饰光斑。

参考原则：Ant Design 暗色模式强调舒适性和信息一致性，暗色下不应打破原有信息层级；本项目应沿用这个原则，只调整视觉 token 和布局层级，不改业务含义。

## 页面范围

第一阶段覆盖用户侧：

- `/projects` 项目工作台
- `/projects/[id]` 项目详情页
- `/projects/[id]/agent` 和 `/projects/[id]/agents`
- `/image` 生图工作台
- `/video` 视频提示词审核台
- `/prompts` 提示词中心
- `/assets` 我的素材
- `/canvas/[id]` 画布工作区

暂不覆盖：

- 后台 `/admin/**`
- 登录页深度改版
- 文档页或 API 路由

## 布局体系

### 全局工作区壳

`ProjectWorkspaceShell` 作为用户侧工作区主壳：

- 左侧导航保留，但改成更克制的暗色面板。
- 导航选中态改为蓝色柔背景，不再使用高亮 cyan。
- 工作区背景使用统一 `shellBg`，去掉页面私有大渐变。
- 页面内容区统一内边距、最大宽度和滚动方式。

### 页面标题区

所有工作区页面使用同一类标题区：

- 左侧：页面标题、短说明、必要面包屑。
- 右侧：主操作、导入、批量操作。
- 搜索和筛选放在标题区下方或同一行右侧，避免页面中部重复大搜索框。

### 内容容器

- 数据密集页面使用全宽面板，不使用大量营销式卡片。
- 卡片仅用于项目、素材、提示词等重复对象。
- 表格/列表优先使用低对比边框和紧凑行高。
- 空状态保持简洁，只说明当前无内容和主要下一步。

## 画布专项

### 工具重排

画布工具分为三层：

1. 左侧创建栏：选择、文本、图片、视频、音频、配置、上传、素材、生产流程。
2. 底部快捷栏：撤销、重做、外观、删除选中、清空或更多。
3. 节点悬浮条：信息、编辑、下载/存素材、生成主操作、更多、删除。

低频节点操作进入“更多”菜单：

- 加白审核
- 更新素材引用
- 截帧
- 续写
- 裁剪
- 多角度
- 锁比例
- 字号调整
- 替换上传

### 视觉规则

- 画布背景统一 `#101217`。
- 网格线降低透明度。
- 选中节点使用信号蓝描边。
- 工具栏使用 `panelBg`、细边框、轻阴影。
- 顶部栏、生产包条、缩放控件、右键菜单共享同一浮层样式。

## 组件规则

### Button

- 主按钮使用信号蓝，不使用 cyan。
- 次按钮使用深色面板背景和边框。
- 危险按钮默认只在确认弹窗、更多菜单或选中态中出现。
- 图标按钮保持 32-36px，工具栏按钮保持稳定尺寸。

### Modal / Drawer

- 使用 `colorBgElevated` 和 `borderSubtle`。
- 标题区和 footer 不做重阴影。
- Drawer 内部优先分 section，不再堆大卡片。

### Card

- 圆角 8px。
- 低对比边框。
- 少阴影。
- 媒体卡片可有图片主视觉，但文字区保持密集可扫。

### Tag / Filter

- 筛选标签使用蓝色柔背景表示选中。
- 状态标签按语义色区分：生成中蓝、完成绿、警告黄、失败红。

## 实施阶段

### 阶段一：设计系统底座

修改：

- `web/src/lib/app-theme.ts`
- `web/src/app/globals.css`
- `web/src/lib/canvas-theme.ts`
- `web/src/app/(user)/projects/project-workspace-shell.tsx`

目标：

- 统一 AntD token。
- 统一 Tailwind CSS 变量和 `studio-*` 工具类。
- 统一用户侧工作区背景、面板、边框、主色。

### 阶段二：用户侧主页面

修改：

- `/projects`
- `/projects/[id]`
- `/image`
- `/video`
- `/prompts`
- `/assets`

目标：

- 页面标题区统一。
- 搜索/筛选统一。
- 卡片、列表、空状态统一。
- 移除页面私有硬编码 `cyan/slate/stone/teal` 视觉分支。

### 阶段三：画布工具

修改：

- `canvas-toolbar.tsx`
- `canvas-tool-button.tsx`
- `canvas-node-hover-toolbar.tsx`
- `canvas-zoom-controls.tsx`
- 必要时微调 `canvas-client-page.tsx`

目标：

- 拆分左侧创建栏和底部快捷栏。
- 节点悬浮工具条改为常用操作 + 更多菜单。
- 缩放、右键菜单、生产包条统一浮层样式。

### 阶段四：后台基础同步

修改：

- `app-theme.ts`
- `/admin/**` 必要的页面级样式

目标：

- 后台共享基础 token。
- 不做沉浸式创作台，只保持清晰、密集、克制。

## 不做事项

- 不改变业务逻辑。
- 不改变本地持久化结构。
- 不重写页面组件结构。
- 不新增大型状态管理。
- 不在第一阶段处理浅色主题的完整重设计。
- 不一次性重构所有 Drawer 内部复杂表单。

## 验收标准

- 用户侧主要页面看起来属于同一产品。
- 暗色主题无明显 stone/cyan/teal 混杂。
- 画布工具不再拥挤，图片/视频节点悬浮条长度明显降低。
- 搜索、筛选、主操作在各页面位置一致。
- AntD Modal、Drawer、Button、Select、Table 与页面背景协调。
- 不引入业务行为变化。
