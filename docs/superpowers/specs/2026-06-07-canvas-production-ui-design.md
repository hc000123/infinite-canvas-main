# 画布承接站与分镜检查时间线 UI 设计

## 背景

当前画布已经承载文本、图片、视频、音频、生成配置、素材、剧本、分镜、Brief、Agent 和任务追踪。问题不再是入口不足，而是主界面信息层级偏散：画布、助手、工作台、抽屉和任务状态同时出现时，用户很难判断当前应该读什么、改什么、下一步做什么。

新的产品主线已经从“先画布后剧本”调整为“项目基准设定 -> 单集剧本导入 -> 导演 Agent 分析 -> 人工确认 -> 服化道 / 美术设计 -> 分镜 / Seedance 提示词 -> 画布承接”。因此画布不再是前置生产主入口，而是承接已确认结果的中转装配站。

本次设计采用“画布承接站 + 分镜检查时间线”的方向：保留无限画布的自由编排能力，把已经审核通过的分镜、提示词、参考素材和视频配置导入为可检查、可微调、可执行生成的节点组。

## 目标

- 让画布成为 AI 短剧生产结果的中转装配站，用于承接、检查、微调和执行生成。
- 把“已审核分镜 / 提示词 / 参考素材 -> 画布节点组 -> 图片 / 视频生成 -> 成品回流素材库”组织成清楚的承接链路。
- 用户输入、AI 输出、提示词、脚本和分镜描述必须完整可读，不默认缩略。
- 系统内置提示、模型过程、运行记录和调试信息默认隐藏，只保留设置与查阅入口。
- 保持当前浅色 / 深色主题能力，优先沿用 `canvasThemes`、`useThemeStore` 和 Ant Design token。

## 非目标

- 不重写画布拖拽、缩放、连线、框选、撤销重做等底层交互。
- 不新增大型状态管理方案。
- 不改变节点、素材、分镜、Agent 或任务日志的数据结构。
- 不把系统推理过程、原始 JSON、规范读取细节放到主操作界面。
- 不在画布内重新承载项目基准设定、剧本导入、导演分析、服化道拆解等前置生产入口。
- 不做移动端完整重构，第一版仍以桌面端为主。

## 上游入口边界

画布只接收上游确认后的结果，不负责生产链路的最初判断。

### 项目基准设定

项目创建时建立全剧前提，用于后续导演分析和服化道判断：

- 作品类型：男频、女频、通用或其他。
- 画面形式：真人、动画、漫改、二次元、写实插画等。
- 画幅和规格：横屏、竖屏、方屏、分辨率、fps、默认时长。
- 视觉风格：年代、色调、镜头质感、参考风格。
- 类型标签：短剧、悬疑、逆袭、甜宠、复仇、都市等。
- 受众和平台倾向。
- 偏好和禁忌：例如电影感、不要太网感、不要夸张表演等。

### 单集导入

剧本按集导入，每集形成独立输入包：

- 本集标题。
- 本集剧情设定或前情摘要。
- 本集核心冲突。
- 本集重点人物。
- 本集爽点、情绪点和反转点。
- 本集剧本文本。
- 用户对本集的特别要求。

### 导演分析与人工确认

导演 Agent 读取项目基准设定和本集输入包，只做信息分析，不直接写入画布、不生成素材、不触发扣费。用户确认导演分析方向正确后，才进入服化道、美术设计和分镜提示词阶段。

### 画布承接时机

画布在上游至少完成以下任一结果后介入：

- 已审核的分镜头表。
- 已审核的 Seedance 提示词草案。
- 已确认的角色、场景、道具和参考素材。
- 需要执行生成的视频配置节点草案。

承接入口分为两类：

- 自由承接画布：只有参考素材、视频配置草案或零散生成结果时进入，保持普通画布体验，不强制展开分镜检查时间线。
- 分镜承接画布：存在已审核分镜头表或已审核 Seedance 提示词时进入，默认提供底部分镜检查时间线，并按镜头聚焦节点组。

## 设计方向

### 整体布局

主界面分为六个稳定区域：

1. 顶部项目栏：项目名、本集名、承接状态、返回本集生产流程、导出、设置、用户状态。
2. 左侧窄导航：画布节点、素材、模板、任务、设置等承接侧入口，图标优先。
3. 中央无限画布：节点、连线、分组和生成链路仍是主要操作区。
4. 右侧上下文检查器：选中节点时看完整输入输出，未选中时显示当前承接批次概览。
5. 底部分镜检查时间线：展示镜头卡片、审核状态、生成状态、时长、缩略图和当前链路。
6. 轻量任务队列：放在底部右侧抽屉，展示进度、失败、重试和必要费用提示。

### 视觉风格

- 基调：温和浅色生产台，搭配深色主题增强版。
- 背景：低对比网格或点阵，避免抢文字。
- 面板：低阴影、轻透明、清晰边界，避免大面积玻璃模糊。
- 强调色：使用少量青色表示 AI / 生成状态，朱红或琥珀表示风险、失败、待确认。
- 工具栏：继续极简扁平，减少胶囊感和厚重阴影。
- 节点：减少装饰，突出内容、状态和连接关系。

## 文字显示规则

这是本次设计的硬约束。

### 必须完整可读

以下内容默认不能用省略号隐藏：

- 用户输入的提示词。
- AI 输出的回答、改写结果和提示词。
- 剧本文本、分镜描述、镜头提示词。
- 生图 Brief 和视频生成提示词。
- 生成配置节点里最终发送给模型的输入预览。

### 避免重复展示

- 同一段提示词只在一个主阅读区完整展示。
- 画布节点展示内容本身，右侧检查器展示该节点的完整编辑和元信息。
- 时间线只展示镜头标题、状态和缩略图，不重复塞完整提示词。

### 长文本处理

- 文本节点优先自动换行，并允许节点高度在合理范围内随内容增长。
- 画布节点不省略关键内容，但超长文本进入节点内阅读区，避免单个节点撑爆画布。
- 右侧检查器承担完整阅读主视图，适合查看长脚本、长提示词、长输出和引用详情。
- 长文本节点可提供“专注阅读 / 收起”状态，但收起后不能丢失访问入口。
- 右侧检查器使用分区阅读：输入、输出、引用、设置、记录。
- 特别长的系统记录放到“记录”Tab，不在主内容区直接展示。

## 系统信息降噪规则

主界面默认只呈现用户需要判断的结果：

- 当前输入是什么。
- 生成出了什么。
- 是否成功、失败、待确认。
- 下一步能做什么。

以下内容默认折叠到“设置 / 记录 / 任务详情”：

- 系统提示词。
- Agent 规范读取清单。
- 模型请求参数的完整 JSON。
- 原始接口响应。
- 中间推理过程。
- debug 字段、stageId、verification、readPaths 等过程字段。

如果系统内容影响用户决策，只展示可读摘要，例如“缺少参考图”“任务失败，可重试”“费用仅为本地估算”。

## 操作逻辑

### 默认自由画布模式

用户进入画布后，默认看到中央画布和底部工具栏：

1. 新建文本、图片、视频、音频或配置节点。
2. 通过连线建立引用关系。
3. 选中节点后，右侧检查器展示完整内容和可用动作。
4. 点击生成后，任务进入底部右侧队列。
5. 生成结果回到画布节点，并可加入素材或进入下一步。

### 分镜承接模式

当画布绑定项目和集数时，底部分镜时间线可展开：

1. 选择镜头卡片。
2. 画布聚焦到该镜头相关节点组。
3. 右侧检查器显示该镜头的完整脚本、Brief、提示词、参考素材和生成结果。
4. 用户检查引用、提示词和配置是否符合上游审核结果。
5. 用户确认后生成图片或视频。
6. 视频完成后自动关联尾帧，支持续写下一镜头。
7. 成品素材回流素材库，并保留项目、集数、镜头和节点来源。

### 承接批次导入

从本集生产流程进入画布时，画布应以“承接批次”为单位接收上游结果：

1. 分镜头表导入为镜头卡片和分镜文本节点。
2. Seedance 提示词导入为视频配置节点和提示词节点。
3. 角色、场景、道具参考导入为素材引用节点。
4. 已有生成结果导入为图片 / 视频素材节点。
5. 用户可以在导入预览里取消某些项，再确认写入画布。

导入过程不会自动生成图片或视频，也不会自动扣费。

### 右侧检查器状态

- 未选中节点：显示当前承接批次概览、缺口、待确认项和返回本集生产流程入口。
- 选中文本节点：显示完整文本、改写 / 生图动作、引用关系。
- 选中图片节点：显示图片、完整提示词、来源、保存素材、裁剪、多角度。
- 选中视频节点：显示视频、完整提示词、首尾帧、续写、任务状态。
- 选中配置节点：显示完整输入预览、模型、尺寸、数量、引用顺序。
- 选中镜头卡片：显示镜头脚本、分镜描述、Brief、提示词和生成结果。

## 组件改造范围

### 一期

- 新增或改造 `CanvasContextInspector`：统一承载节点详情、完整输入输出、承接批次概览和 AI 助手辅助状态。
- `CanvasToolbar`：整理为更轻的底部工具 dock，只保留高频动作，低频项放入更多菜单。
- `CanvasNode`：调整文本节点和配置节点的内容展示规则，避免关键文字缩略。
- `CanvasNodeHoverToolbar`：减少常驻文字按钮，改为图标 + tooltip，复杂动作交给右侧检查器。
- `CanvasAssistantPanel`：改造成右侧上下文面板的辅助状态，不再总是抢占主界面。
- 分镜相关组件：抽取底部分镜检查时间线所需的镜头卡片和状态摘要。
- 本集生产流程到画布的写入入口：统一命名为“创建承接画布”或“导入到画布”，避免出现“新建一集画布并导入剧本”等前置绑定文案。

### 二期

- 增加顶部命令搜索，统一“新增节点、打开素材、跳转镜头、查找节点”等动作。
- 增加节点组 / 镜头组视觉聚焦。
- 增强任务队列，把生图、视频、Agent 文本任务合并展示。
- 增加专注阅读模式，用于查看长脚本、长提示词和长输出。

## 验收标准

- 打开画布后，用户能一眼分清画布、分镜、检查器和任务队列。
- 任意用户输入或 AI 输出都能在画布节点或右侧检查器中完整查看。
- 主界面不直接展示系统推理过程、原始 JSON 或规范读取噪音。
- 选中节点后，下一步动作明显，不需要在多个抽屉之间寻找。
- 绑定集数的画布能通过底部时间线按镜头检查、微调和执行生成。
- 未绑定集数的普通画布仍能保持自由节点创作体验。
- 项目基准设定、剧本导入、导演分析和服化道拆解不会重新出现在画布主界面作为前置入口。
- 从本集生产流程创建承接画布时，只绑定当前集并承接结果，不自动生成图片或视频。

## 待确认

- 底部分镜时间线是否默认展开，还是只在绑定项目 / 集数后展开。
- 右侧检查器默认宽度是否固定为 420px，还是允许用户拖拽记忆。
- 文本节点自动增高的上限是多少，超过后是否进入节点内滚动。
- 是否需要单独提供“只看用户内容 / 显示记录”的全局开关。
- 承接批次导入时，是否需要强制用户逐项确认素材引用和视频配置节点。
